## 数据质量报告

### Excel 格式识别

所有 13 个文件均使用**日历网格格式**（calendar grid）：
- 日期行：一行 7 列，每列一个日期，对应周一至周日
- 内容行：紧跟日期行下方，对应列包含当日活动内容
- 特殊情况：`大二上+寒假.xlsx` 使用周历格式（星期一~日表头，无具体日期），日期由学期范围推算

### 状态检测

| 状态 | 数量 | 说明 |
|------|------|------|
| in_progress | ~1,550 | 无特殊颜色标记（默认状态） |
| completed | 3 | 灰色背景单元格 |
| incomplete | ~4 | 红色字体（仅 2026 年文件） |

> **注意**：大部分条目无颜色标记，默认归为 `in_progress`。实际上许多历史活动应为已完成状态，但 Excel 中未用颜色区分。

### 缺失数据

- **无时间信息**：所有 Excel 导入条目均无 `start_time`、`end_time`、`duration`
- **无分类信息**：所有条目 `category` 为空，未关联预定义分类
- **2019 年数据稀疏**：仅 68 天有记录（大二上学期，可能是开始记录的学期）

### 重复数据处理

- `博一下+暑假.xlsx` 与 `博一下+暑假+博二上（202401-202412）.xlsx` 存在大量重叠
- 增量导入机制通过 `(date + content + source)` 三元组去重，成功跳过 187 条重复

### 日期准确性

- 公式日期文件（如 `大三上工作计划.xlsx`）：使用 `data_only=True` 正确解析
- 学期推算日期（`大二上+寒假.xlsx`）：基于入学时间 2018.9 和学期映射表推算，精度为±1周
- 年份校正：通过 `SEMESTER_YEAR_MAP` 将文件名中的学期映射到正确年份

## 已知问题

1. **状态标记不完整**：绝大多数条目为 `in_progress`，无法准确反映历史任务的完成情况
2. **缺少分类**：Excel 数据未包含分类信息，需后续手动或自动分类
3. **大二上日期为近似值**：周历格式文件的日期由学期范围按周推算，可能偏差 1-7 天
4. **无时间维度**：Excel 仅记录每日活动内容，无具体时段信息

## 改进建议

1. **自动分类**：基于活动内容关键词（如"论文"→科研、"课程"→学习）自动归类
2. **状态回填**：对历史数据批量标记为 `completed`（合理假设历史任务已完成）
3. **内容去重合并**：同一天多条记录可考虑合并展示
4. **数据补充**：通过 CLI 或 Web Dashboard 补充缺失的分类和时间信息
